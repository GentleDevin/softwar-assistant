## Git Commit Message 规则

### 格式要求
```
<类型>(<模块>): <简短描述>

<详细说明>
```

### 类型定义
- **fix**: 修复bug
- **feat**: 新增功能
- **refactor**: 代码重构（不改变功能）
- **docs**: 文档更新
- **style**: 代码格式调整
- **test**: 测试相关
- **chore**: 构建/工具/配置更新

### 示例
```
fix(ui): 修复检索进展时间显示延迟问题

- 步骤显示时立即显示时间，不再延迟
- 缩短更新间隔至0.3秒，提升流畅度
```

### 规则
1. 标题不超过50字符
2. 描述精炼，突出核心修改
3. 使用中文描述
4. 多项修改用 `-` 列出
